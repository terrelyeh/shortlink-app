環境變數設定指南

部署至 Vercel Hobby Plan 所需的完整環境變數說明

在 Vercel 哪裡設定？

  1. 登入 Vercel Dashboard
  2. 選擇你的 ShortLink 專案
  3. 左側選 Settings
  4. 點擊 Environment Variables
  5. 逐一新增以下變數，Environment 選擇 Production + Preview + Development
1

DATABASE_URL

必填
是什麼

PostgreSQL 資料庫的連線 URI。Prisma ORM 透過這個連線字串存取所有資料：使用者帳號、短網址、點擊記錄、Campaign 等。

為什麼需要

這是整個應用的核心資料來源。沒有 DATABASE_URL，應用無法啟動，所有頁面都會回傳 500 錯誤。

如何設定（Supabase）
  1. 登入 Supabase Dashboard
  2. 選擇你的 Project（或建立新的，區域選 ap-southeast-1 離台灣最近）
  3. 左側選 Project Settings → Database
  4. 往下找到 Connection string 區塊，選 URI tab
  5. 複製整段連線字串，把 [YOUR-PASSWORD] 替換成 database password
# 格式範例
postgresql://postgres.[project-ref]:[password]@aws-0-ap-southeast-1.pooler.supabase.com:6543/postgres

首次部署後，需要推送 Schema 到資料庫：

npx prisma db push
注意事項
⚠ 使用 Transaction mode (port 6543) — Vercel serverless 環境請用 pooler 連線，不要用 direct connection，否則會很快耗盡連線數。
⚠ 密碼只顯示一次 — database password 只在建立時顯示，忘記的話需要在 Supabase Dashboard 重設。
2

AUTH_SECRET

必填
是什麼

NextAuth.js 用來加密 session token、CSRF token 和 cookie 的密鑰。一組隨機的 Base64 字串。

為什麼需要

沒有 AUTH_SECRET，NextAuth 無法管理使用者登入狀態。使用者無法登入，所有受保護的 Dashboard 頁面都無法存取。

如何設定

在終端機執行：

openssl rand -base64 32

# 輸出範例：
K7x2Qf3mNp8Lj1Rv5Yt9Wz4Xa6Bc0De2Fg7Hi3Jk=

把輸出直接貼到 Vercel 的 Value 欄位。

注意事項
⚠ 更改會登出所有人 — 如果你更換 AUTH_SECRET，所有使用者的 session 都會失效，需要重新登入。
⚠ 不要用簡單字串 — 一定要用 openssl rand 產生的隨機值，不要手打 "my-secret" 之類的。
3

GOOGLE_CLIENT_ID + SECRET

必填
是什麼

Google OAuth 2.0 的憑證，讓使用者可以用 Google 帳號登入。CLIENT_ID 是公開的識別碼，CLIENT_SECRET 是伺服器端的密鑰。

為什麼需要

ShortLink 目前只支援 Google OAuth 登入。沒有這兩個值，登入按鈕會失效，使用者無法進入系統。

如何設定

Step 1 — 設定 OAuth Consent Screen（首次才需要）

  1. 前往 Google Cloud Console，選擇或建立 Project
  2. 左側 APIs & Services → OAuth consent screen
  3. User Type 選 External，填寫 App name 和 email
  4. 其他全部跳過，一路 Save and Continue 到完成

Step 2 — 建立 OAuth Client ID

  1. 左側 Credentials → + CREATE CREDENTIALS → OAuth client ID
  2. Application type 選 Web application
  3. Authorized JavaScript origins 加入你的 domain
  4. Authorized redirect URIs 加入（兩個都要）：
# 本地開發
http://localhost:3000/api/auth/callback/google

# Vercel Production
https://你的專案名.vercel.app/api/auth/callback/google
  1. 點 Create，複製彈窗中的 Client ID 和 Client Secret
注意事項
⚠ Redirect URI 必須完全匹配 — 包含 protocol、domain、path，少一個字元都會導致 "redirect_uri_mismatch" 錯誤。自訂 domain 也要另外加。
⚠ Testing 狀態限制 — OAuth consent screen 在 "Testing" 時，只有手動加入的測試帳號能登入。要讓所有人登入，需要點 PUBLISH APP。
⚠ CLIENT_SECRET 不可外洩 — 只能放在 Vercel 環境變數或 .env.local，絕對不要 commit 到 Git。
4

NEXT_PUBLIC_APP_URL

必填
是什麼

應用的公開 URL，用於 NextAuth callback、email 連結等。NEXT_PUBLIC_ 前綴表示會暴露在前端 JS 中。

為什麼需要

NextAuth 需要知道完整 URL 才能正確處理 OAuth redirect。設錯會導致登入後 redirect 到錯誤網址。

如何設定
環境值
本地開發http://localhost:3000
Vercel（預設）https://你的專案名.vercel.app
自訂 domainhttps://sl.mycompany.com
注意事項
💡 不要加結尾斜線 /。必須包含 https:// protocol。
5

NEXT_PUBLIC_SHORT_URL

必填
是什麼

短網址在 UI 上顯示的完整前綴，也用於 QR Code 生成和複製連結功能。

為什麼需要

使用者在 Dashboard 看到的短網址如 sl.mycompany.com/s/abc123，其中 sl.mycompany.com/s 部分就來自這個變數。不設定的話 UI 會顯示預設佔位符。

如何設定
環境值
本地開發http://localhost:3000/s
Vercel（預設）https://你的專案名.vercel.app/s
自訂 domainhttps://sl.mycompany.com/s
注意事項
💡 結尾不加 /。必須包含 /s 路徑，因為所有短網址都在 /s/[code] 路徑下。
6

IP_HASH_SALT

必填
是什麼

一組隨機的 hex 字串，用來對使用者 IP 做 SHA-256 雜湊。系統不儲存原始 IP，只儲存雜湊值，用於點擊去重和獨立訪客計算。

為什麼需要

隱私保護 — GDPR 等隱私法規要求不能直接儲存 IP。加了 salt 的 SHA-256 雜湊可以：判斷同 IP 是否重複點擊（去重）、計算獨立訪客數，但無法反推原始 IP。

如何設定
openssl rand -hex 32

# 輸出範例：
a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2
注意事項
⚠ Production 必填 — 沒設定的話，點擊記錄功能會報錯。Redirect 本身仍正常，但不會有任何分析數據。
⚠ 設定後不要更改 — Salt 改了之後同一個 IP 會產生不同的 hash，導致去重失效、獨立訪客數突然暴增。
⚠ 不可外洩 — 如果 salt 洩漏，攻擊者可以用彩虹表反推 IP 地址。
7

ALLOWED_EMAIL_DOMAIN

選填
是什麼

限制只有特定 email 網域的 Google 帳號才能登入。例如設成 mycompany.com，只有 xxx@mycompany.com 的帳號能登入。

為什麼需要

如果你的 ShortLink 是公司內部工具，不想讓外部人員用個人 Gmail 登入，就需要這個限制。不設定 = 所有 Google 帳號都能登入。

如何設定
# 只填 domain，不要加 @
mycompany.com
注意事項
⚠ 設錯 domain 會鎖死所有人 — 確認拼寫正確。設錯需要到 Vercel Dashboard 修改後重新部署。
💡 目前只支援單一 domain。如果需要多 domain 支援，需修改程式碼 src/lib/auth.ts。
